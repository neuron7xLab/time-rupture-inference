# SBOM Contract

`sbom.spdx.json` is **SPDX 2.3 JSON**, generated by
`scripts/generate_sbom.py` deterministically from
`requirements-ci.lock` + `pyproject.toml`. It is NOT hand-written and
NOT a fabricated document.

## Guarantees

- Standard SPDX-2.3 format (`spdxVersion`, `creationInfo.created`,
  `documentNamespace` with source commit, `packages`, DEPENDS_ON
  `relationships`).
- Package set == the pinned `requirements-ci.lock` set, plus the
  project root package. No package absent from the lock; none missing.
- Includes runtime deps (numpy, matplotlib, pyyaml) and CI/dev tools
  (pytest, ruff, mypy, types-PyYAML) — exactly what the lock pins.

## Not claimed

Licenses are `NOASSERTION` (not audited here). No vulnerability data.
No file-level analysis (`filesAnalyzed: false`). The SBOM is an
inventory, not a security clearance.

## Enforcement

`scripts/generate_sbom.py --verify-only` (CI / `audit_release_trust`)
fails on any drift between the SBOM and the lock, on a non-SPDX-2.3
document, or on a missing required dependency.
