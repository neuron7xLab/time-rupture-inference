# SLSA Level Declaration

## Achieved: SLSA Build L2 (and not above)

The release workflow produces a signed build-provenance attestation
via `actions/attest-build-provenance` (keyless OIDC, GitHub-hosted
runner). By the SLSA v1.0 model this is **Build L2**: provenance
exists, is authenticated, and is generated by the hosted build
service — but NOT by an isolated/reusable SLSA generator.

**Not L3.** No isolated builder, no reusable SLSA generator path is
used. Any "SLSA L3" claim here would be false. This document and
`scripts/audit_release_trust.py` forbid an L3 claim without a verified
isolated mechanism.

## Mechanism

- Build: `python -m build` (sdist + wheel) on a GitHub-hosted runner,
  AFTER the full test gate + frozen runner.
- Provenance: `actions/attest-build-provenance@<pinned-SHA>`,
  `subject-path: dist/*`, `id-token: write`.

## Verification command

```
gh attestation verify dist/<artifact> \
  --repo neuron7xLab/time-rupture-inference
```

(See `docs/RELEASE_VERIFICATION.md`.)

## Not claimed

Not SLSA L3. Not hermetic. Not reproducible-build-verified. Not a
guarantee the source is free of vulnerabilities. The attestation binds
*who/what/where built it*, not *that the code is safe*.

## Next level

L3 would require an isolated/reusable SLSA generator
(e.g. slsa-framework/slsa-github-generator) and is a deliberate future
step, not done here.
