# Value Positioning

This repository is best evaluated as a reproducible research-systems
artifact, not as a productized platform. The value is the integration
discipline, not novelty of any single part.

## 1. What this artifact is

A fail-closed falsification apparatus: redacted hypothesis spec →
opaque probe → adversarial falsifier battery → sealed verdict →
evidence ledger → preserved negatives → human-gated next experiment.

## 2. What this artifact is not

Not a product, not a benchmark leaderboard, not an autonomous research
agent, not a claim generator, not a safety guarantee.

## 3. What it proves about the author

Capacity to build an evaluation system that resists its own
optimism: pinned falsifiers, preserved REDs, machine-checked claim
boundaries, reproducible frozen evidence, and a supply-chain trust
layer — engineered, tested, and CI-gated, not asserted.

## 4. What it does not prove about the science

No real-world validity, no domain generality, no biological fidelity,
no learned-model superiority. The surviving positives are narrow and
scoped (`docs/SPEC.md` S1/S2); the dominant product is the sealed
negative lineage.

## 5. Why it is expensive to rebuild

The cost is not lines of code; it is the disciplined negative space:
nine preserved lineages, an information-theoretic boundary proof
(v8.3), a re-derived valid benchmark (v8.4), and a machine-checkable
trust layer that blocks the cheap path of polishing claims.

## 6. Current evidence basis

Frozen v4/v5 numbers byte-identical across all commits; sealed
v1..v9 lineage; 391 tests; ruff + mypy --strict + claims-lint +
provenance + supply-chain aggregate green. See
`docs/CLAIM_SOURCE_MATRIX.md` and `evidence/DOC_TRUST_AUDIT.json`.

## 7. Current value blockers

- Independent reproduction is OPEN (`GAP_1`).
- Domain breadth is OPEN (`GAP_2`).
- Real-world validity is not established.
- Biological fidelity is not claimed.
- Product readiness is blocked by explicit governance, by design.

## 8. Next milestones that increase value most

1. One external, author-absent reproduction with a valid proof bundle
   (closes `GAP_1`).
2. ≥2 structurally distinct task families with pinned falsifiers
   (closes `GAP_2`).
3. A second surviving positive on a non-synthetic family.

## 9. Buyer / reviewer fit

Fits: a research-engineering hiring committee; an AI-evaluation lab; an
R&D team needing private hypothesis falsification; a collaborator
assessing author capability. Does not fit: a buyer expecting a
turnkey hosted service.
